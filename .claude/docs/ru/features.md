# Функциональность

## Обновление портфеля
**Триггер**: Пользователь нажимает кнопку "Обновить" на экране PortfolioScreen
**UseCase**: `UpdatePortfolioUseCase`
**Шаги**:
1. Валидация API-ключей (CheckSettingsUseCase)
2. Получение цен всех тикеров с MEXC
3. Получение балансов аккаунта MEXC (запрос с подписью HMAC)
4. Фильтрация исключённых монет и позиций с нулевой стоимостью
5. Сохранение в Room через PortfolioRepository

## Ребалансировка
**Триггер**: Пользователь нажимает кнопку "Ребалансировать"
**UseCase**: `RebalancerUseCase` (алгоритм из 4 шагов)
**Шаги**:
1. Параллельное получение топ-N монет с CMC и торгуемых пар с MEXC
2. Построение списка доступных монет (CMC ∩ MEXC − исключённые)
3. Продажа монет, не входящих в доступный список
4. Покупка отсутствующих доступных монет (по среднему размеру позиции)
5. Ребалансировка существующих: продажа перевешенных, покупка недовешенных

## Частичная продажа
**Триггер**: Пользователь открывает панель продажи, вводит сумму в USDT, нажимает "Продать"
**UseCase**: `SellUseCase`
**Логика**: Пропорциональная продажа — доля каждой монеты = её вес в общей стоимости портфеля
**Минимум**: Пропуск монет, где сумма продажи < $1

## Настройки
**Экран**: SettingsScreen с 5 редактируемыми полями
**Хранилище**: `EncryptedSharedPreferences` (AES256-GCM) — API-ключи НИКОГДА не хранятся в открытом виде
**Поля**: CMC API Key, MEXC API Key, MEXC API Secret, Лимит топ монет, Исключённые монеты

## Правила хранения ключей
- API-ключи → ТОЛЬКО в `EncryptedSharedPreferences`
- Используется `MasterKeys.getOrCreate(MasterKeys.AES256_GCM_SPEC)` (API security-crypto 1.0.0)
- Имя файла: `crypto_secure_prefs`
